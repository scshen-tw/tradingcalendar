========================================================
  TradingCalendar 專案 — Outlook cbas 自動更新說明
  (給新 Claude 視窗的完整 context)
========================================================

【專案位置】
  本機：d:\VS Code\TradingCalendar\
  GitHub：https://github.com/scshen-tw/tradingcalendar
  GitHub Pages：https://scshen-tw.github.io/tradingcalendar/calendar.html

【專案目的】
  自動從 Outlook 郵件擷取 CB（可轉債）競拍資料，
  並合併本機股票競拍 JSON，產生互動式行事曆 HTML，
  推送至 GitHub Pages 供瀏覽器直接開啟。

========================================================
  一、資料來源
========================================================

1. CB 資料（可轉債）
   來源：Outlook 資料夾「cbas」中，主旨含「cb案件整理表」的最新一封郵件
   格式：郵件內嵌 HTML 表格
   欄位：承銷方式 / CB代碼 / 發行標的 / TCRI / 掛牌日 / 競拍截止 / 發行金額 / 轉換價

2. 股票競拍資料
   來源：d:\VS Code\Auction\auction_stocks.json（另一個專案 auction-viewer 產生）
   GitHub raw URL（GitHub Actions 環境用）：
     https://raw.githubusercontent.com/scshen-tw/auction-viewer/main/auction_stocks.json
   欄位：證券代號 / 證券名稱 / 投標結束日 / 撥券日期 / 實際承銷價格 / 取消競價拍賣

========================================================
  二、輸出格式
========================================================

events.json — 所有行事曆事件的陣列，每筆格式：
  {
    "date":       "YYYY-MM-DD",
    "type":       "CB競拍截止" | "CB競拍掛牌" | "CB詢圈掛牌" | "股票競拍截止" | "股票競拍掛牌",
    "code":       "債券或股票代號",
    "name":       "名稱",
    "tcri":       "tcri5" | "有擔" | ""  (CB才有，股票留空),
    "amount":     "10.00E"              (CB才有，股票留空),
    "conv_price": "485"                 (CB轉換價 / 股票承銷價，未定留空),
    "display":    "行事曆顯示文字"
  }

calendar.html — 靜態單頁行事曆，EVENTS_DATA 直接嵌入在 JS 裡：
  // __EVENTS_START__
  const EVENTS_DATA = [...];
  // __EVENTS_END__

========================================================
  三、程式檔案說明
========================================================

extract_outlook.py（主程式）
  - 連線 Outlook COM，搜尋 cbas 資料夾
  - 找到最新一封主旨含「cb案件整理表」的郵件
  - 解析郵件 HTML 表格（支援 rowspan/colspan）
  - 辨識欄位（COLUMN_PATTERNS 模糊比對，容錯多種表頭格式）
  - 判斷承銷方式（競拍 → 截止+掛牌 兩筆; 詢圈 → 只有掛牌）
  - Outlook 失敗時 fallback：讀現有 events.json 保留 CB，只更新股票
  - 最後呼叫 extract_stocks.extract_stock_events() 合併股票事件
  - 輸出 events.json 與 calendar.html

extract_stocks.py（股票競拍擷取模組）
  - 優先讀本機 auction_stocks.json
  - 本機檔不存在時改從 GitHub raw URL 下載（供 GitHub Actions 使用）
  - 篩選：截止日 >= 今天 - 14天
  - 跳過已取消競拍（「取消競價拍賣」欄位有值）
  - 每筆產生兩個事件：股票競拍截止 + 股票競拍掛牌（有掛牌日才加）

update_stocks.py（獨立股票更新器）
  - 不需要 Outlook，只更新股票部分
  - 讀現有 events.json 保留所有 CB 事件
  - 重新呼叫 extract_stocks.extract_stock_events()
  - 合併後寫回 events.json 與 calendar.html

auto_update.bat（本機 CB 完整更新）
  - 執行 extract_outlook.py（需要 Outlook 開啟且有新郵件）
  - 成功後 git add / commit / push

update_stocks.bat（本機股票獨立更新）
  - 執行 update_stocks.py（不需要 Outlook）
  - 成功後 git add / commit / push

.github/workflows/update.yml（GitHub Actions 自動排程）
  - 每天 UTC 01:00（台灣時間 09:00）自動執行
  - 可手動觸發（workflow_dispatch）
  - 執行 update_stocks.py（從 GitHub 下載 auction_stocks.json）
  - 有變更才 commit，避免空 commit

========================================================
  四、COLUMN_PATTERNS 欄位比對關鍵字
========================================================

  承銷方式：  承銷方式、方式、銷售方式
  CB代碼：    CB代碼、代號、債代、標的代號、編號、債券代號
  名稱：      發行標的、名稱、債名、標的名稱、公司名、債券名稱
  TCRI：      tcri、TCRI、評等、TCRI評等、信評
  掛牌日：    掛牌日、掛牌、上市日、掛牌日期
  競拍截止：  競拍期間、詢圈期間、競拍截止、拍止、截止日、競拍日、投標截止
  發行金額：  發行金額、金額、總額、發行規模
  轉換價：    轉換價、轉換價格、換股價、轉換

  （如郵件表格欄位名稱改變，在此處新增關鍵字即可）

========================================================
  六、常見問題與處理方式
========================================================

Q: Outlook 連不上或找不到 cbas 資料夾？
A: extract_outlook.py 會 fallback 讀現有 events.json 的 CB 事件，
   只更新股票部分，不會中斷整個流程。

Q: 郵件表格欄位名稱對不上？
A: 執行時會印出「全部欄位標題」與對應結果，
   在 COLUMN_PATTERNS 補上新的關鍵字即可。

Q: 新股票沒更新到 calendar.html？
A: auction_stocks.json 更新後，執行 update_stocks.bat 即可；
   或等 GitHub Actions 當天 09:00 自動跑。

Q: 手動觸發 GitHub Actions？
A: 進 https://github.com/scshen-tw/tradingcalendar/actions
   點「每日更新股票競拍行事曆」→「Run workflow」
